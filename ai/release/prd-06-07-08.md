## 2. Pré-requisitos de deploy

1. **Firestore Rules** atualizadas precisam ser publicadas:
   `npm run deploy:rules` (inclui `system_logs`, `notifications`, `notificationPreferences`, avatar/nickname).
   ⚠️ Sem isso, criação de logs/notificações e edição de avatar falham com `permission-denied`.
2. **Índices Firestore** (se exigidos): queries novas usam `where`+`orderBy`:
   - `notifications`: `where userId == X` + `where type == Y` + `orderBy createdAt desc`
   - `system_logs`: `where type == Y` + `orderBy createdAt desc`
   Firebase indicará no console o link de criação do índice composto na 1ª execução. Criar antes do uso real.
3. **≥1 usuário admin** (`role: "admin"`) semeado para acessar a Administração (promoção fora de banda).
4. Variáveis de ambiente Firebase já configuradas (sem mudança nesta entrega).

## 3. Rollout sugerido

1. Publicar Rules + índices (passo 2/1) em staging.
2. Deploy do build (App Hosting) em staging.
3. Smoke/UAT manual (ver `/local-env` → limites): login admin, aprovar usuário (→ log + notificação),
   trocar senha, upload avatar, preferências, badge do sino.
4. Promover para produção após UAT.

## 4. Riscos e mitigação

| Risco | Mitigação |
|---|---|
| Rules não publicadas → escrita falha | Publicar `deploy:rules` ANTES do deploy do app |
| Índice composto ausente → query de notificações/logs falha em runtime | Criar índices via link do console na 1ª query (staging) |
| Status API mostra placeholder | Esperado (D-A3); comunicar que telemetria é fase futura |
| Notificações Ranking/Jogos não chegam | Esperado (D-A5); só Sistema no V1 |
| Avatar base64 perto de 1MB/doc | `imageToDataUrl` comprime < ~700KB + barra imagem grande |
